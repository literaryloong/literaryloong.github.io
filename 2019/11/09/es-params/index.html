ES 参数说明

创建时间:2019-11-09 17:18
阅读:

  2. refresh操作
  • Local Gateway
    1. ElasticSearch常用配置及性能参数

    • flush操作

    • index.translog.flush_threshold_ops : 执行多少次操作后执行一次flush，默认无限制
    • index.translog.flush_threshold_size : translog的大小超过这个参数后flush，默认512mb
    • index.translog.flush_threshold_period : 多长时间强制flush一次,默认30m
    • index.translog.interval : es多久去检测一次translog是否满足flush条件
    • index.translog.sync_interval : 控制translog多久fsync到磁盘,最小为100ms
    • index.translog.durability : translog是每5秒钟刷新一次还是每次请求都fsync , 有2个取值:
      • request (每次请求都执行fsync,es要等translog fsync到磁盘后才会返回成功)
      • async(默认值,translog每隔5秒钟fsync一次)

    refresh操作

    index.refresh_interval : 刷新间隔

    Local Gateway

    • gateway.expected_nodes

      群集中应预期的（数据或主节点）节点数。本地分片的恢复将在预期节点数加入群集后立即开始。默认值为0

    • gateway.expected_master_nodes

      群集中预期的主节点数。本地分片的恢复将在预期的主节点数加入群集后立即开始。默认值为0

    • gateway.expected_data_nodes

      群集中预期的数据节点数。本地分片的恢复将在预期的数据节点数加入群集后立即开始。默认值为0

    • gateway.recover_after_time

      如果未达到预期的节点数，则恢复过程将等待配置的时间量，然后再尝试恢复。不管如何。如果配置了其中一个设置，则默认为 。5m``expected_nodes

    持续时间超时后，只要满足以下条件，恢复就会开始：recover_after_time

    • gateway.recover_after_nodes

      只要这么多数据或主节点已加入群集，就恢复。

    • gateway.recover_after_master_nodes

      只要这么多主节点已加入群集，就恢复。

    • gateway.recover_after_data_nodes

      只要这么多数据节点已加入群集，就恢复。
